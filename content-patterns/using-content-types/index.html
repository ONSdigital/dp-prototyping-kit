When and how to use content types

You can use bulletins or articles to write about statistics. Bulletins, articles and datasets are grouped and published in releases.

Bulletins

A bulletin gives headline numbers and commentary alongside essential quality and methodology information.

It should be released with new data, linking to that data and to more detailed content for users who want more than an overview.

Bulletins should focus on topics that we know users are interested in. If we know users are interested in a number of topics in a release, we can publish more than one bulletin in a "release".

Where there's a clear user need for more detail on a specific topic (gender pay gap, for example), a release can contain more than one bulletin
A bulletin should give users A bulletin should not contain
An overview of the main findings, and first commentary on newly released data In-depth commentary
Links to the new data, and to specific parts of the data which might be useful Deep technical analysis
A summary of the most important quality and methodology information, linking to more detailed documents where necessary Detailed explanations of upcoming changes, roadmaps, or transformation plans

Overview bulletins

Where a release contains more than one bulletin, an overview summarises the most important information from across the whole release.

For example, the Crime release may include bulletins on Weapons, Fraud and Homicide.

The Crime overview would comment on overall Crime trends in the latest data, and might briefly explain the main causes of any changes (linking to the relevant bulletins for more detail).

Articles

An article is less structured than a bulletin, and allows for more detailed information on a subject. Typical articles might include:

  • deeper analysis of a topic that cannot fit in the bulletin template
  • analysis of the latest data alongside other data sources
  • more information about methodology or planned changes to a future release

As with any content on our website, there should be a clear user need for everything we publish. The length and detail of an article should be relative to the interest of our users. We will measure this using analytics and through user research.

An article should only be used when it is not possible to communicate all of the information users need in a bulletin (because of the number of words required, or because the analysis crosses multiple bulletins or releases, for example).

How to decide whether you need more bulletins

Digital publishing will help you determine whether more pages would add value. We will use search data and user research to understand what users need. We may run extra research, or suggest small-scale experiments we can run to validate the need for a new page.

We will also consider:

  • whether users might expect to find information about this topic anywhere else (on another page in this release, or elsewhere on the ONS website, for example)
  • how the user need relates to the time and effort required to create and publish extra content
  • can we communicate this using anything that already exists
  • does it need to be published on the day of release
  • whether this content is best written as an article or a bulletin

Once the page has been published, we will continually monitor its use to understand whether there is a continuing user need for this content.
